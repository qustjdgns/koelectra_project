# KoELECTRA 활용 지도 앱 리뷰 감성 분석  
## 리뷰 만족도와 시장 점유율의 역설(Paradox) 분석

---

## 1. 개요

### 1-1. 문제 정의: "만족도 = 점유율" 가설에 대한 질문

본 프로젝트는 대한민국 지도 앱 시장의 양대 산맥인 네이버 지도와 카카오맵의 사용자 리뷰를 KoELECTRA 모델로 분석하는 것을 목표로 한다.

직관적으로,  
"사용자 만족도(긍정 리뷰)가 높은 앱이 더 많은 사용자(시장 점유율)를 확보할 것이다"라고 가정할 수 있다.  

하지만, 과연 이 가설은 사실일까?

본 프로젝트는 이 질문에서 시작한다. 두 플랫폼의 방대한 리뷰 데이터를 감성 분석하여 **사용자 만족도**와 **실제 시장 점유율**이 과연 일치하는지, 혹은 두 지표 사이에 예상치 못한 **역설(Paradox)**이 존재하는지를 규명하고자 한다.

---

### 1-2. 지도 앱 시장의 중요성

지도 애플리케이션은 단순한 길찾기 도구를 넘어, 맛집 검색, 장소 예약, 리뷰 확인 등 소비자의 의사결정에 직접적인 영향을 미치는 '슈퍼 앱'으로 진화하고 있다. 따라서 사용자의 리뷰 감성과 실제 사용 행태를 분석하는 것은 플랫폼의 성패를 가르는 중요한 요소이다.

### 2-1. 데이터 구성

두 데이터셋은 동일한 4개의 열(Column)로 구성되어 있다.

| 컬럼명 | 데이터 타입 | 설명 | 모델 활용 |
|--------|--------------|------|------------|
| at | Object (datetime) | 리뷰가 작성된 날짜와 시간 | X (분석에만 사용) |
| score | Integer | 사용자가 부여한 원본 평점 (1~5점) | X (분석에만 사용) |
| document | Object (String) | 사용자가 작성한 리뷰 원문 텍스트 | O (Input) |
| label | Integer | 긍/부정 라벨 (0: 부정, 1: 긍정) | O (Target) |

`label`은 `score`를 기준으로 사전에 처리된 값이다.  
(예: 1~2점 → 0, 4~5점 → 1)

KoELECTRA 모델은 이 중 document를 입력(Input)으로 받아 label을 예측(Target)하도록 학습된다.

## 3. "역설"의 발견: 실제 시장 점유율 분석

리뷰 감성(Label) 분석 결과만 보면 카카오맵이 시장을 주도해야 하지만,  
실제 시장 데이터는 정반대의 결과를 보여주며, 역설이 발생한다.

### 3-3. 사례 2: "승자독식 생태계 (Lock-in Effect)"

네이버는 **검색 → 블로그/플레이스 → 지도**로 이어지는 생태계를 통해  
사용자를 잃지 않고 유지하고 있다.  

즉, 네이버 지도 점유율은 앱 자체 만족도보다는  
플랫폼 생태계의 연동성(Lock-in Effect)에서 비롯된다.

| 강점 | 깔끔한 UI/UX, 도보 및 자전거 길찾기 | 풍부한 데이터베이스, 네비게이션, 검색 연동 |

## 4. KoELECTRA 모델 성능 및 결과

데이터 분석과 시장 분석을 통해, 두 플랫폼의 데이터가 본질적으로 다름을 확인하였다.  
이에 따라 각각의 데이터셋에 대해 개별 모델을 파인튜닝(Fine-tuning)하였다.

### 4-3. 결과 분석: '특화 모델'의 타당성

- **koelectra_small_naver (85.1%)**  
  네이버 특유의 불만과 칭찬이 혼재된 U자형 분포를 학습.  
  새로운 불만 리뷰(VOC)를 85% 정확도로 탐지 가능.

- **koelectra_small_kakao (90.9%)**  
  카카오맵의 긍정적 리뷰 중심 J자형 분포를 학습.  
  젊은층의 긍정적 표현을 91% 정확도로 분류.

## 5. KoELECTRA 모델링 전략의 타당성

### 5-1. 두 개의 모델을 학습시킨 이유

만약 두 플랫폼 데이터를 합쳐 하나의 모델로 학습했다면,  
상반된 데이터 특성(부정 편향 vs 긍정 편향)이 서로의 학습을 방해했을 것이다.  

그 결과, 85~91% 수준의 개별 모델보다 낮은 정확도를 보였을 가능성이 크다.

---

### 5-2. 프로젝트의 접근 방식

본 프로젝트는 두 개의 개별 모델을 구축하였다.

- koelectra_small_naver: 네이버의 부정 편향(U자형 분포) 데이터 학습  
- koelectra_small_kakao: 카카오맵의 긍정 편향(J자형 분포) 데이터 학습  

이 접근은 각 데이터의 역사적 배경과 특성을 반영하며, 모델이 실제 상황을 학습하도록 설계되었다.

## 6. 결론 및 제언

### 결론

본 프로젝트는 "사용자 만족도가 높으면 시장 점유율도 높을 것이다"라는 가설에서 출발하였다.  
하지만 분석 결과, **리뷰 만족도(카카오맵 우세)** 와 **시장 점유율(네이버 우세)** 은 일치하지 않는 역설이 존재함을 확인했다.

KoELECTRA 모델을 통한 감성 분석에서,  
네이버 모델은 85.1%, 카카오 모델은 90.9%의 높은 정확도를 보였다.  
이는 각각의 특화 모델 접근이 타당함을 입증한다.

- 카카오맵은 UI·UX 및 대중교통 중심 기능 강점을 기반으로 젊은층 타깃을 유지할 필요가 있다.
