## 1. 개요

### 1-1. 문제 정의: "만족도 = 점유율" 가설에 대한 질문

본 프로젝트는 대한민국 지도 앱 시장의 양대 산맥인 네이버 지도와 카카오맵의 사용자 리뷰를 KoELECTRA 모델로 분석하여,
“사용자 만족도(긍정 리뷰)가 높은 앱이 더 많은 사용자(시장 점유율)를 확보할 것”이라는 가설이 실제로 성립하는지 검증하고자 한다.

직관적으로, 만족도가 높은 앱이 점유율도 높을 것으로 예상할 수 있으나, 실제 데이터는 예상치 못한 **역설(Paradox)**이 존재함을 확인했다. 본 프로젝트는 이 역설의 근본적인 원인을 텍스트 마이닝 기법을 통해 정량적으로 입증하고자 한다.

---

### 1-2. 지도 앱 시장의 중요성

지도 애플리케이션은 단순한 길찾기 도구를 넘어, 맛집 검색, 장소 예약, 리뷰 확인 등 소비자의 의사결정에 직접적인 영향을 미치는 '슈퍼 앱'으로 진화하고 있다. 따라서 사용자의 리뷰 감성과 실제 사용 행태를 분석하는 것은 플랫폼의 성패를 가르는 중요한 요소이다.

### 2-1. 데이터 구성

두 데이터셋은 동일한 4개의 열(Column)로 구성되어 있다.

| 컬럼명 | 데이터 타입 | 설명 | 모델 활용 |
|--------|--------------|------|------------|
| at | Object (datetime) | 리뷰가 작성된 날짜와 시간 | X (분석에만 사용) |

| label | Integer | 긍/부정 라벨 (0: 부정, 1: 긍정) | O (Target) |

## 3. "역설"의 발견: 실제 시장 점유율 분석 및 원인 증명

리뷰 감성 분석 결과만 보면 **카카오맵이 시장을 주도해야** 하지만, **실제 시장 데이터는 정반대의 결과** 를 보여주며, **역설이 발생** 합니다.

---

### 3-1. 실제 시장 점유율 비교

2024년 12월 시장조사에 따르면, 국내 지도 앱 시장 점유율은 다음과 같습니다:

| 플랫폼     | 시장 점유율 |
|------------|-------------|
| 네이버 지도 | 68.3%       |
| 카카오맵    | 31.2%       |

> **결론:** 리뷰 만족도(카카오맵 우세)와 시장 점유율(네이버 우세)은 일치하지 않는 **역설**이 존재합니다.

### 3-2. 역설을 유발하는 주요 원인 가설

본 프로젝트는 이 역설을 설명하기 위해 다음 **3가지 원인 가설**을 설정했습니다. 이 가설들은 3-3절의 토픽 모델링을 통해 증명됩니다.

| 구분   | 원인 가설                    | 내용 |
|--------|-----------------------------|------|
| 가설 1 | **승자독식 생태계** (Lock-in Effect) | 네이버 점유율은 앱 만족도보다는 **검색 → 블로그/플레이스** 연동성 때문 |
| 가설 2 | **지도 난민(대안재) 효과**         | 카카오맵의 긍정 리뷰는 **네이버의 불만으로 인해 생긴 반사 효과** |
| 가설 3 | **다른 타겟, 다른 기능**          | 사용자층과 기능이 다르고, 네이버 사용자들은 **핵심 기능 문제를 구체적으로 서술**하는 경향 |

### 3-3. 토픽 모델링을 통한 원인 가설 증명

## 4. KoELECTRA 모델 학습 과정

데이터 분석과 시장 분석을 통해, 두 플랫폼의 데이터가 본질적으로 다름을 확인하였다.  
이에 따라 각각의 데이터셋에 대해 개별 모델을 파인튜닝(Fine-tuning)하였다.

### 4-1. 모델 선정

KoELECTRA (koelectra-base-v3-discriminator) 모델을 사용함.

이는 한국어 리뷰 텍스트의 미묘한 뉘앙스를 잘 이해하도록 사전에 학습된 고성능 언어 모델임.

### 4-2. 토큰화 (Tokenization)

**"텍스트 → 숫자"** 로 변환하는 과정임.

모델이 이해할 수 있도록 "편리해요" 같은 한글 리뷰를 [14502, 12222] 같은 숫자 배열(토큰)로 변환.

max_length=256으로 설정하여 모든 리뷰의 길이를 동일하게 맞춤.

### 4-3. 데이터 구성 (Train/Validation Split)

전체 데이터를 8:2 비율로 분리함.

학습 데이터 (80%): 모델 학습을 위한 데이터.

검증 데이터 (20%): 모델의 실제 성능을 확인하기 위한 검증용 데이터.

### 4-4. 학습 및 검증 (Train & Validation)

학습 (Train): 모델이 32개(배치 크기)의 데이터를 처리하고, 정답과 비교하여 "오차(Loss)"를 계산. 이 오차를 줄이는 방향으로 모델을 업데이트함. (4 에포크 반복)

검증 (Validation): 1 에포크 학습 종료 후, "검증 데이터"로 시험을 수행하며, 이 점수가 Validation Accuracy (최종 정확도)가 됨.

- **koelectra_small_kakao (90.9%)**

  젊은층의 긍정적 표현을 91% 정확도로 분류.

## 6. KoELECTRA 모델링 전략의 타당성

### 6-1. 두 개의 모델을 학습시킨 이유

만약 두 플랫폼 데이터를 합쳐 하나의 모델로 학습했다면,  
상반된 데이터 특성(부정 편향 vs 긍정 편향)이 서로의 학습을 방해했을 것이다.  

그 결과, 85~91% 수준의 개별 모델보다 낮은 정확도를 보였을 가능성이 크다.

---

### 6-2. 프로젝트의 접근 방식

본 프로젝트는 두 개의 개별 모델을 구축하였다.

- koelectra_small_naver: 네이버의 부정 편향(U자형 분포) 데이터 학습  
- koelectra_small_kakao: 카카오맵의 긍정 편향(J자형 분포) 데이터 학습  

이 접근은 각 데이터의 역사적 배경과 특성을 반영하며, 모델이 실제 상황을 학습하도록 설계되었다.

---

## 7. 결론

본 프로젝트는 "사용자 만족도가 높으면 시장 점유율도 높을 것이다"라는 가설에서 출발하였다.  
하지만 분석 결과, **리뷰 만족도(카카오맵 우세)** 와 **시장 점유율(네이버 우세)** 은 일치하지 않는 역설이 존재함을 확인했다.

KoELECTRA 모델을 통한 감성 분석에서,  
네이버 모델은 85.1%, 카카오 모델은 90.9%의 높은 정확도를 보였다.  
이는 각각의 특화 모델 접근이 타당함을 입증한다.
